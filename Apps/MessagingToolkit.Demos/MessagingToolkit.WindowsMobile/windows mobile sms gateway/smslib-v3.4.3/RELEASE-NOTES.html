SMSLib v3.4.3 (Oct 13, 2009)
Release Notes

Upgrading from v3.4.2

There are some compatibility issues that need to be addressed! Check the following list of fixes for details.

Enhancements / Fixes

  1. The IP Modem Gateway now starts with the BINARY protocol set as default.
  2. The IP Modem Gateway now supports secure IP connections (#237).
  3. Fixed error in Wavecom AT Handler (#115).
  4. Annoying debug error appearing in log (#248).
  5. Incorrent thread shutdown, leading to stopService() call being stalled.
  6. Compatibility Issue: The xxxInboundNotification() methods have been renamed to xxxInboundMessageNotification().
  7. Compatibility Issue: The xxxOutboundNotification() methods have been renamed to xxxOutboundMessageNotification().
  8. Decoupling of notification methods. Events are still serialized, but the invokation code is moved outside the main processing thread (#251).
  9. Now using prepared statements for frequent SQLs in the SMSServer/DB interface (#247).
  10. Possible Compatibility Issue: International originator numbers in inbound messages are now prefixed with "+" (#253).
  11. Changed the detection of storage locations in the default AT handler (#255).
  12. Null pointer exception upon failure to start gateway (#256).
  13. Handling incomplete multipart messages: New logic added for identifying "orphaned" messages and a new callback handler has been introduced. (#254).
  14. Clickatell handler was broken in latest release. (#267).
  15. Implementation of the CMMS command which increases the GSM throughput. (#257).

That's it!

For more information, visit the SMSLib Web Site.
